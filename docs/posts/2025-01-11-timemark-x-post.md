# TimeMark：让时间变得可感知

不知道从什么时候开始，我喜欢在每个月的第一天查看日历，看看这一年又过去了多少。

有时候是突如其来的焦虑——原来2025年已经过去了87.36%，有时候是温柔的提醒——还好，这个月还有三分之一的时间可以用来做些什么。

这种感觉让我开始思考：如果能把看不见的时间流逝变成看得见的进度条，会不会让我们更珍惜当下？

于是就有了这个小小的项目：**TimeMark**——一个极简的时间进度可视化工具。它不是在制造焦虑，而是想帮助我们与时间建立更温和的关系。

年度进度、月度进度、周进度，甚至是人生进度（假设80年），每一个数字背后都是对生命的一份敬畏。

---

## 产品背后的思考

在做这个项目的时候，我反复问自己：我们真的需要另一个进度条吗？

后来我发现，重要的不是进度条本身，而是它代表的那种态度。当你看到这一年还剩128天的时候，你不会想要更加珍惜吗？当你知道这周已经过了5天的时候，你不会更想好好利用周末吗？

这就是我设计的初衷：**让时间变得可感知，让每一刻都有重量**。

产品的细节里藏着很多小心思：三种不同形态的进度条（线性、圆形、弧形），完全自定义的配色，暗黑模式的支持，还有那些在特定进度时出现的温柔提醒——10%时说"开始就是成功的一半"，50%时说"恰好半程，继续前进"，90%时说"最后冲刺，别放弃"。

所有的数据都存储在本地，因为我觉得，关于时间的数据，应该只属于你自己。

## 现代 AI 辅助开发的实践

作为开发者，我最近开始尝试拥抱 **Vibe Coding** 的理念。

这次的开发过程特别有意思：我用 **Grok** 分析了几个竞品，让它帮我输出产品需求文档，然后整个开发过程都在 **Claude Code** 中完成。

从需求分析到代码实现，全程 AI 辅助。这种感觉有点神奇——你不再是孤独的编码者，而是和一个永远在线、知识渊博的搭档一起创造。它会建议更好的组件结构，会提醒我处理边界情况，甚至会主动推荐适合的配色方案。

这个项目的每一行代码，都凝结着人类创意和 AI 辅助的协作。Next.js + TypeScript 让技术基础很扎实，Tailwind CSS 让样式调整变得即时有趣。当我把进度条的动画调到恰到好处的流畅时，那种满足感依然是无与伦比的。

我把这个项目开源，就是想分享这种新的开发方式：**你不需要完美的计划，但你需要合适的工具**。代码在 GitHub，结构清晰易懂。更重要的是，如果你也有一个想做很久的小项目，试试让 AI 成为你开发伙伴吧。

## 时间的意义

TimeMark 现在静静地运行在 GitHub Pages 上，每天都会有一些人来访，看看时间的进度。

我不知道它是否真的改变了什么，但我知道，创作它的过程改变了我。

我们常常说"时间不够用"，但其实时间一直都在那里，只是我们没有真正看见它。当你把时间可视化，当你意识到这一年还剩 100 多天，这个月还剩 10 天，你也许会做出不同的选择。

**不是要让我们焦虑地追赶时间，而是要让我们温柔地珍惜时间。**

如果你也想看看时间的进度，欢迎访问。如果你是开发者，欢迎看看代码，或许能给你一些启发。如果你也有一个小小的想法，别让它只停留在脑海里——用 Vibe Coding 的方式，让它成为现实吧。
